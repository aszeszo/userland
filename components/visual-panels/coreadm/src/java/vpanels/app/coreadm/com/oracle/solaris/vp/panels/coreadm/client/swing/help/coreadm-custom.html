Settings for Core File Management

You can further configure core file management by clicking the “View details / customize “button on the Core File settings tab. Next select a tab to select configuration changes for global or for per-process core files. The following configuration changes can be made:

  • Enable generation of global or per-process core files depending on the tab selected.

  • Define the path and core name file pattern that each core file should be stored in. You can insert embedded variables into the path using the insert arrow to the right of the Path. See Customizing the Core File Name Pattern for more information.

  • Whether to allow set-id or set-gid core files to use the global or per-process core pattern.

  • Select the contents of the core file. See Customizing the Contents of the Core File for more information.

    • Add or delete content by moving entries into or out of the Selected screen

    • Reset content to the system default by clicking the Defaults button
