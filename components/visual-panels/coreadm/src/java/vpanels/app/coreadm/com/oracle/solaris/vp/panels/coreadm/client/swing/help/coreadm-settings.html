Settings for Core Files

When the Settings tab is selected, you can configure whether core files are generated and how the files are stored. Two types of core files can be created by the system. A per-process core file is owned and can be examined by the owner of the process that terminated. A global core file is owned by root. The contents of a per-process and a global core file for an event can be identical, depending on the content options that are selected.

The following pre-set configuration schemes are available:

  • Do not generate any core files

  • Store the core files in the current directory

  • Store the core files in the current directory with a name including the application name and the process ID

  • Store core files in global location

If none of the pre-set schemes are sufficient, a custom scheme can be created. See Settings for Core File Management for more information about how to customize core file management.
